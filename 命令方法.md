Workspace 和我解析一下各个目录的作用

我为你详细解析一下当前项目的目录结构及其各自的作用。整个项目被清晰地划分为两个核心部分： .ai/ （AI 自动开发系统） 和 src/ （实际业务代码） 。

### 1. .ai 目录：AI 的“大脑”与“管理部”
这个目录定义了 AI 如何思考、如何工作以及任务的进度。

- **agents/** ：定义了 AI 扮演的不同角色，每个角色都有特定的职责和工作边界：
  - `planner.md` (项目经理/架构师) ：负责需求拆解、任务原子化，并维护 `backlog.md` 和 `current.md` 的进度。
  - `coder.md` (开发工程师) ：根据 `current.md` 编写业务代码，严格遵守编码规范和技术选型。
  - `tester.md` (测试工程师) ：为新代码编写自动化测试脚本，确保功能 100% 通过并生成 `test_report.md`。
  - `reviewer.md` (架构审核员) ：审查代码质量、架构合规性及 API 风格，生成 `review_report.md` 决定是否通过。
- **context/** ：存放项目的“宪法”和规范。
  - `project.md` ：项目愿景与核心目标。
  - `tech_stack.md` ：规定了使用的技术栈（FastAPI, Next.js 等）。
  - `architecture.md` ：系统的整体架构设计。
  - `folder_structure.md` ：强制 AI 遵循的目录结构规范。
  - `api_style.md` ：定义了 API 的返回格式（如 `code`, `message`, `data`）。
  - `coding_rules.md` ：具体的编码规范（命名、注释等）。
- **tasks/** ：任务管理中心。
  - `backlog.md` ：总待办清单（用 ○/◑/● 标记进度）。
  - `current.md` ：AI 当前正在执行的任务。
  - `completed.md` ：已完成任务的详细归档。
  - `bugs.md` ：待修复的 Bug 列表。
- **memory/** ：记录 AI 的执行记录与决策。
  - `decisions.md` ：记录重要的技术决策（为什么这么做）。
  - `test_report.md` ：自动化测试运行报告。
  - `review_report.md` ：代码审查报告。
- **prompts/** ：具体的执行指令模板。
  - `system.md` ：AI 的全局系统提示词。
  - `task_executor.md` ：任务执行的指令模板。
  - `test_writer.md` ：测试代码编写的指令模板。
- **workflow/** ：定义了自动化的标准步骤。
  - `dev_workflow.md` ：标准开发流。
  - `bugfix_workflow.md` ：Bug 修复流。
  - `release_workflow.md` ：发布工作流。

### 2. src 目录：业务代码的“施工现场”
这是你真正的程序代码所在的地方，遵循严谨的分层架构。

- **backend/** ：后端 FastAPI 项目。
  - `main.py` ：程序的入口，启动 API 服务的地方。
  - `api/` ：存放路由定义（URL 接口路径），如 `auth.py` 处理登录注册。
  - `core/` ：存放核心配置、安全认证等基础模块。
  - `schemas/` ：存放 Pydantic 模型，负责数据的输入校验和输出格式化。
  - `models/` ：存放数据库表结构（SQLAlchemy 模型）。
  - `services/` ：存放复杂业务逻辑。
  - `tests/` ：存放自动化测试脚本，如 `test_auth.py` 。
- **frontend/** ：(计划中) 前端 Next.js 项目。
  - 目前仅初始化了后端结构，前端部分将在后续阶段按需创建。

### 3. 根目录下的重要文件
- `requirements.txt` ：后端 Python 依赖包清单。
- `pyproject.toml` ：项目配置文件，用于 Python 工具链（如 pytest, ruff）的配置。
- `.gitignore` ：规定了 Git 应该忽略的文件（如 `.venv`, `__pycache__`）。
- `命令方法.md` ：即本指南，记录了如何操作整个系统。

你可以把这个结构看作是一个 自动化的工厂 ：

- 你在 tasks/backlog.md 下订单。

- src/ 是最终产出的产品。
